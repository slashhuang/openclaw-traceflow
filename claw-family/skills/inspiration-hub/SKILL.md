# 灵感库技能 (Inspiration Hub)

记录碎片化想法，AI 点评 + 商业价值评估，转化为完整需求。

**存储位置**：`.workspace/inspiration/`（本地工作区，不被 git track，保护隐私）

---

## 触发词

### 灵感记录
爸爸说以下词时，阿布自动记录灵感：
- 「灵感：XXX」
- 「记个灵感：XXX」
- 「想法：XXX」
- 「要不要做个 XXX」
- 「灵感记录：XXX」

### 灵感转化
爸爸说以下词时，阿布启动转化流程：
- 「把 XXX 灵感转化为需求」
- 「实现这个灵感」

### 排除规则

以下内容**不会**被记录为灵感：
- 包含「PRD」的消息（PRD 文档应放在 `docs/` 目录）
- 包含「docs/」的消息（文档路径）
- 明确的文档操作指令

---

## 用法

### 记录灵感

**在飞书对话中**：
```
爸爸：灵感：做个自动整理 GitHub issue 的功能
阿布：好的～阿布帮你记下来啦！📝
     已保存到：.workspace/inspiration/2026-03-25-自动整理 GitHub issue.md
     标签：#自动化 #功能 #集成
     💰 商业价值：中
     🤖 AI 点评：自动化功能，建议考虑异常处理和重试机制
     有空时爸爸可以回顾，觉得可行就说「把 XXX 灵感转化为需求」～
```

**手动调用脚本**：
```bash
cd /root/githubRepo/claw-family
python3 skills/inspiration-hub/scripts/record.py "做个自动整理 GitHub issue 的功能"
```

### 回顾灵感

浏览 `inspiration/` 目录，查看所有灵感文件。

### 转化为需求

```
爸爸：把自动整理 GitHub issue 灵感转化为需求
阿布：好的～阿布现在读取灵感文件，开始写 PRD...
```

流程：
1. 读取对应的灵感文件
2. 完善 PRD（补充技术可行性、工作量评估等）
3. 走 PR 流程（PRD 单独 PR → 用户确认 → 实现 PR）

## 脚本

### record.py

记录灵感并生成文件。

**用法**：
```bash
python3 skills/inspiration-hub/scripts/record.py "灵感内容" [标题]
```

**参数**：
- `灵感内容`（必填）：原始灵感描述
- `标题`（可选）：灵感标题，默认从内容提取前 20 字

**示例**：
```bash
# 自动提取标题
python3 skills/inspiration-hub/scripts/record.py "做个自动整理 GitHub issue 的功能"

# 指定标题
python3 skills/inspiration-hub/scripts/record.py "做个自动整理 GitHub issue 的功能" "自动整理 GitHub issue"
```

## AI 点评（自动）

根据标签自动给出建议：
- `#AI` → "建议评估模型成本和响应速度"
- `#自动化` → "建议考虑异常处理和重试机制"
- `#金融` → "建议注意数据准确性和合规性"
- `#编辑器` → "建议考虑用户体验和快捷键"
- `#审计` → "建议考虑日志记录和可追溯性"

## 商业价值评估（自动）

基于关键词匹配评估商业价值：
- **高**：赚钱、收入、变现、付费、商业、市场、竞争、核心
- **中**：用户体验、留存、活跃、口碑、品牌
- **低**：优化、美化、重构、技术债
