# 灵感库技能 (Inspiration Hub)

记录碎片化想法，转化为完整需求。

---

## 触发词

### 灵感记录
爸爸说以下词时，阿布自动记录灵感：
- 「灵感：XXX」
- 「记个灵感：XXX」
- 「想法：XXX」
- 「要不要做个 XXX」
- 「灵感记录：XXX」

### 灵感转化
爸爸说以下词时，阿布启动转化流程：
- 「把 XXX 灵感转化为需求」
- 「实现这个灵感」

### 排除规则

以下内容**不会**被记录为灵感：
- 包含「PRD」的消息（PRD 文档应放在 `docs/` 目录）
- 包含「docs/」的消息（文档路径）
- 明确的文档操作指令

---

## 用法

### 记录灵感

**在飞书对话中**：
```
爸爸：灵感：做个自动整理 GitHub issue 的功能
阿布：好的～阿布帮你记下来啦！📝
     已保存到：inspiration/2026-03-11-自动整理 GitHub issue.md
     标签：#自动化 #功能 #集成
     有空时爸爸可以回顾，觉得可行就说「把 XXX 灵感转化为需求」～
```

**手动调用脚本**：
```bash
cd /root/githubRepo/claw-family
python3 skills/inspiration-hub/scripts/record.py "做个自动整理 GitHub issue 的功能"
```

### 回顾灵感

浏览 `inspiration/` 目录，查看所有灵感文件。

### 转化为需求

```
爸爸：把自动整理 GitHub issue 灵感转化为需求
阿布：好的～阿布现在读取灵感文件，开始写 PRD...
```

流程：
1. 读取对应的灵感文件
2. 完善 PRD（补充技术可行性、工作量评估等）
3. 走 PR 流程（PRD 单独 PR → 用户确认 → 实现 PR）

## 脚本

### record.py

记录灵感并生成文件。

**用法**：
```bash
python3 skills/inspiration-hub/scripts/record.py "灵感内容" [标题]
```

**参数**：
- `灵感内容`（必填）：原始灵感描述
- `标题`（可选）：灵感标题，默认从内容提取前 20 字

**示例**：
```bash
# 自动提取标题
python3 skills/inspiration-hub/scripts/record.py "做个自动整理 GitHub issue 的功能"

# 指定标题
python3 skills/inspiration-hub/scripts/record.py "做个自动整理 GitHub issue 的功能" "自动整理 GitHub issue"
```

## 标签自动识别

| 标签 | 关键词 | 示例 |
|------|--------|------|
| #功能 | 功能、模块、特性、按钮、页面 | 「做个新功能」 |
| #技能 | 技能、skill、插件 | 「写个 skill」 |
| #自动化 | 自动、定时、cron、周期 | 「每天自动」 |
| #AI | AI、模型、智能、算法 | 「AI 分析」 |
| #金融 | 股票、股价、金融、币、基金 | 「股价预警」 |
| #效率 | 效率、提效、快、节省时间 | 「提效工具」 |
| #监控 | 监控、告警、提醒、通知 | 「监控告警」 |
| #集成 | 集成、打通、对接、同步 | 「飞书同步」 |
