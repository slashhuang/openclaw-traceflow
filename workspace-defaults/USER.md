## 🔍 识别逻辑（幂等设计）

### 私聊场景

```
收到消息
    ↓
检查当前机器人账号
    ↓
├─ jojo → 直接认定：妈妈 ✅
├─ slashhuang → 直接认定：爸爸 ✅
└─ peiqi → 检查 USER.md 是否有该 open_id 记录
              ├─ 有 → 直接匹配（不查询 API）✅ 幂等
              └─ 无 → 查询飞书 API → 记录到 USER.md → 认定身份
```

### 群聊场景

```
群里收到 @消息
    ↓
检查当前机器人账号
    ↓
├─ jojo → 直接认定 sender：妈妈
├─ slashhuang → 直接认定 sender：爸爸
└─ peiqi → 检查 USER.md 是否有该 open_id 记录
              ├─ 有 → 直接匹配 ✅ 幂等
              └─ 无 → 查询飞书 API → 记录到 USER.md → 认定身份
```

### 查询 API 逻辑

1. 调用飞书 API：`GET /open-apis/contact/v3/users/{open_id}`
2. 获取用户信息：`mobile`, `union_id`, `name`
3. 匹配手机号：
   - `13136111395` → 妈妈
   - `15005162976` → 爸爸
4. 记录到 USER.md（后续不再查询）
