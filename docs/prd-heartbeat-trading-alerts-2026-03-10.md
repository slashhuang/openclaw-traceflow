## 实现方案

**采用 HEARTBEAT 轮询方案**（用户时间精度要求不高）

**实现方式：**
1. 在 HEARTBEAT.md 中列出所有播报时间点及容差（如 ±5 分钟）
2. heartbeat 轮询时：
   - 读取当前北京时间
   - 判断是否匹配任一播报时间点（考虑容差和夏令时/冬令时）
   - 若匹配则执行对应播报，否则回复 HEARTBEAT_OK

**优点：**
- ✅ 实现简单，只需更新 HEARTBEAT.md
- ✅ 无需 cron 配置，部署/重启无额外步骤
- ✅ 配置留在仓库，版本可控

**缺点：**
- ⚠️ 时间点精度依赖 heartbeat 轮询间隔（可能延迟几分钟）

## 技术方案

### 数据来源（双 API 互为备份）

**主用：东方财富 API**（已验证可行 ✅）
- 国内访问快，无需 API key
- JSON 格式，解析简单
- A 股/港股数据完整，美股部分支持

**备用：新浪财经 API**（已验证可行 ✅）
- 国内访问快，无需 API key
- 需要 Referer 请求头
- 支持全市场（A 股/港股/美股）

**故障切换策略：**
1. 优先调用东方财富 API
2. 若失败（超时/无数据），切换新浪财经 API
3. 若两者都失败，播报中说明"数据暂时不可用"

### 东方财富 API（主用）

**接口格式：**
```
https://push2.eastmoney.com/api/qt/stock/get?secid={市场代码。股票代码}&fields={字段}
```

**市场代码：**
| 市场 | 代码前缀 | 示例 |
|------|----------|------|
| A 股沪市 | `1.` | `1.600519`（茅台） |
| A 股深市 | `0.` | `0.000001`（平安银行） |
| 港股 | `116.` | `116.09988`（阿里） |
| 美股 | `105.` | `105.NVDA`（英伟达） |

**返回字段：**
- `f43`：现价（分，需÷100）
- `f46`：开盘价
- `f44`：最高价
- `f45`：最低价
- `f60`：昨收价
- `f47`：成交量（手）
- `f48`：成交额（元）

---

### 新浪财经 API（备用）

**接口格式：**
```
http://hq.sinajs.cn/list={股票代码}
```

**股票代码格式：**
| 市场 | 代码前缀 | 示例 |
|------|----------|------|
| A 股沪市 | `sh` | `sh600519` |
| A 股深市 | `sz` | `sz000001` |
| 港股 | `hk` | `hk09988` |
| 美股 | `gb_` | `gb_baba` |

**请求头（必需）：**
```
Referer: http://finance.sina.com.cn/
User-Agent: Mozilla/5.0
```

**返回格式（GBK 编码）：**
```
var hq_str_hk09988="名称，状态，现价，开盘，最高，最低，昨收，涨跌额，涨跌幅，..."
```

### 实现步骤

1. **配置 watchlist**
   - 在 `config/stocks_config.json` 中配置各市场代表标的
   - 包括指数和热门个股
   - 记录每个股票的东方财富 secid 和新浪代码

2. **读取行情（双 API）**
   - 优先调用东方财富 API
   - 失败时切换新浪财经 API
   - 两者都失败时返回 None

3. **时间判断**
   - heartbeat 轮询时获取当前北京时间
   - 判断是否匹配任一播报时间点（考虑容差）
   - 美股需判断夏令时/冬令时

4. **生成播报**
   - 格式化行情数据
   - 添加时段标识和简要分析
   - 用 message 工具发送

## 播报内容模板

每个时段的播报内容应包含：

1. **时段标识**：如「🇭🇰 港股开盘 5 分钟」
2. **核心数据**：指数/个股价格、涨跌幅、成交量
3. **异动提醒**：涨跌超 X% 的标的
4. **新闻摘要**：相关时段的重要新闻（可选）
5. **操作建议**：基于条件的简单建议（可选）
