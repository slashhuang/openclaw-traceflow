## 💡 需求背景

基于 6 份炒股最佳实践调研，设计一个**实时盯盘**智能炒股助手 Skill。

**核心痛点**：
- ❌ 手动盯盘累，容易错过买卖点
- ❌ 工作忙，没时间看盘
- ❌ 情绪化交易（追涨杀跌）
- ❌ 没有系统性风控

**核心价值**：
- ✅ **7x24 小时实时监控**（永不疲劳）
- ✅ **毫秒级预警推送**（不错过任何机会）
- ✅ **纪律性交易**（严格执行止损止盈）
- ✅ **多用户支持**（爸爸妈妈独立配置）

### 1. 数据源选择

**核心原则**：无需 API Key，全部免费直接用

| 市场 | 主数据源 | 备用数据源 | 刷新频率 |
|------|----------|------------|----------|
| A 股 | 东方财富 WebSocket | 新浪财经 | 3 秒/次 |
| 港股 | 东方财富 WebSocket | 新浪财经 | 3 秒/次 |
| 美股 | 东方财富 API | 新浪财经 | 5 秒/次 |
| 加密货币 | Binance WebSocket | CoinGecko | 1 秒/次 |

**优点**：
- ✅ 无需 API Key（直接用）
- ✅ 国内访问快（延迟低）
- ✅ 数据稳定（东方财富是专业财经数据商）

### 2. 防限流策略

#### 2.1 数据源防限流

**WebSocket 优先**：
```
❌ REST API 轮询：每秒 1 次 = 3600 次/小时 → 易限流
✅ WebSocket 长连接：1 次连接，持续推送 → 不限流
```

**多数据源备用**：
```
主数据源：东方财富 WebSocket
    ↓ 失败/限流
备用数据源：新浪财经 API
    ↓ 失败/限流
第三数据源：腾讯财经 API
```

**本地缓存 + 增量更新**：
```python
# 只推送变化的数据
if current_price != cache[symbol]['price']:
    if abs(change_pct) >= threshold:
        send_alert()
    cache[symbol]['price'] = current_price
```

#### 2.2 推送防限流

**优先级队列**：
```
┌─────────────────┐
│ 紧急（止损）     │ → 立即推送（<1 秒）
├─────────────────┤
│ 预警（±5%）      │ → 延迟推送（<30 秒）
├─────────────────┤
│ 普通（±3%）      │ → 合并推送（5 分钟）
└─────────────────┘
```

**推送合并**（避免刷屏）：
```
❌ 错误：每只股票涨跌都推送（3 条）
✅ 正确：5 分钟汇总推送（1 条）
   - 持仓股异动汇总：NVDA +3%, AAPL +3%, MSFT +3%
```

**频率限制**：
- 每只股票每日最多推送 10 次
- 单用户每分钟最多推送 5 条
- 静默时段：23:00-07:00（只推送紧急预警）

#### 2.3 自动降级

```
正常模式：3-5 秒刷新
    ↓ 检测到限流（429 错误）
降级模式：30 秒刷新
    ↓ 持续限流
最低模式：5 分钟刷新 + 只监控止损
```

### 5. 核心代码示例

#### 5.1 实时监控循环（带防限流）

```python
import asyncio
import websockets

async def monitor_price(symbol, user_config):
    """实时监控价格（带防限流）"""
    uri = f"wss://push2.eastmoney.com/quote/{symbol}"
    
    # 限流计数器
    daily_alert_count = {symbol: 0}
    last_merge_time = time.time()
    pending_alerts = []
    
    async with websockets.connect(uri) as websocket:
        while True:
            try:
                data = await websocket.recv()
                current_price = data['price']
                change_pct = (current_price - user_config['cost']) / user_config['cost'] * 100
                
                # 检查限流
                if daily_alert_count[symbol] >= 10:
                    continue  # 跳过今日剩余推送
                
                # 检查静默时段
                if is_silent_hours() and alert_type != 'critical':
                    continue  # 只推送紧急预警
                
                # 检查止损（最高优先级）
                if change_pct <= -user_config['stop_loss']:
                    await send_urgent_alert(...)
                    daily_alert_count[symbol] += 1
                    continue
                
                # 推送合并（避免刷屏）
                pending_alerts.append({
                    'symbol': symbol,
                    'change_pct': change_pct
                })
                
                # 每 5 分钟合并推送一次
                if time.time() - last_merge_time > 300:
                    await send_merged_alerts(pending_alerts)
                    pending_alerts = []
                    last_merge_time = time.time()
                
            except websockets.exceptions.ConnectionClosed:
                # 连接断开，切换备用数据源
                switch_to_backup_source()
            
            await asyncio.sleep(3)  # 3 秒刷新
```

#### 5.2 预警优先级队列（带限流保护）

```python
import asyncio

class AlertQueue:
    def __init__(self):
        self.urgent_queue = asyncio.PriorityQueue()  # 紧急
        self.warning_queue = asyncio.PriorityQueue()  # 预警
        self.normal_queue = asyncio.PriorityQueue()   # 普通
    
    async def process_alerts(self):
        while True:
            # 优先处理紧急预警
            try:
                alert = self.urgent_queue.get_nowait()
                await send_immediate(alert)
                continue
            except asyncio.QueueEmpty:
                pass
            
            # 处理普通预警
            try:
                alert = self.warning_queue.get_nowait()
                await send_delayed(alert, delay=5)
                continue
            except asyncio.QueueEmpty:
                pass
            
            await asyncio.sleep(1)
```
