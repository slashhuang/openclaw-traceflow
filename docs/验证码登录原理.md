## 整体架构

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐     ┌─────────────┐
│   富途服务器  │ ←→ │  FutuOpenD   │ ←→ │  Telnet 端口  │ ←→ │  用户/脚本   │
│  (云端)      │     │  (本地进程)   │     │  (22222)     │     │  (验证码)    │
└─────────────┘     └──────────────┘     └─────────────┘     └─────────────┘
       ↑                    ↑
       │                    │
       └──── SMS 验证码 ─────┘
            (手机号)
```

### 核心组件说明

| 组件 | 作用 | 端口/文件 |
|------|------|-----------|
| 富途服务器 | 账号认证、发送验证码、验证登录 | 云端服务 |
| FutuOpenD | 本地交易网关进程 | 本地进程 |
| Telnet 端口 | 验证码输入通道 | 22222 |
| API 端口 | 量化交易 API 接口 | 11113 |
| WebSocket 端口 | 实时行情推送 | 33333 |
| AppData.dat | 登录凭证缓存 | 本地文件 |

## 完整链路详解

### 1. 启动阶段

```
用户执行 ./scripts/start.sh
        ↓
读取 FutuOpenD.xml 配置文件
        ↓
启动 FutuOpenD 进程
        ↓
打开三个端口监听:
  - 11113 (API 端口，给客户端用)
  - 22222 (Telnet 端口，输入验证码用)
  - 33333 (WebSocket 端口，可选)
```

**配置文件关键项** (`FutuOpenD.xml`):

```xml
<!-- 登录账号：手机号格式 +86 13800138000 -->
<login_account>15005162976</login_account>

<!-- 登录密码（明文或 MD5） -->
<login_pwd>Brucelee0716</login_pwd>

<!-- Telnet 端口（启用才能输入验证码） -->
<telnet_port>22222</telnet_port>
```

### 2. 认证触发阶段

```
FutuOpenD 启动后
        ↓
尝试用配置的账号密码登录富途服务器
        ↓
富途服务器检测到:
  - 新设备登录 (OpenD 是独立进程)
  - 或密码登录需要二次验证
        ↓
富途服务器发送 SMS 验证码到手机 (15005162976)
        ↓
FutuOpenD 进入"等待验证码"状态
        ↓
此时进程在运行，但无法提供 API 服务
```

**为什么需要验证码？**

- **风控机制**：防止账号被盗用
- **无人值守场景**：OpenD 是无头进程，需要额外验证
- **设备绑定**：验证码确保你是账号主人

### 3. 验证码输入阶段

```
用户收到短信：【富途】验证码 358167
        ↓
用户执行：./scripts/input-verify-code.sh 358167
        ↓
脚本内部调用 expect:
  spawn telnet 127.0.0.1 22222  → 连接到 Telnet 端口
  send "input_phone_verify_code -code=358167\r" → 发送命令
        ↓
FutuOpenD 接收命令
        ↓
转发给富途服务器验证
        ↓
富途服务器返回：验证成功
        ↓
FutuOpenD 保存登录凭证到 AppData.dat
        ↓
FutuOpenD 正式开始工作（可以处理 API 请求）
```

**Telnet 命令列表**:

```bash
# 连接 Telnet
telnet 127.0.0.1 22222

# 富途定义的命令:
req_phone_verify_code              # 请求发送验证码
input_phone_verify_code -code=xxx  # 输入验证码
req_pic_verify_code                # 请求图片验证码
input_pic_verify_code -code=xxx    # 输入图片验证码
help                               # 查看帮助
relogin                            # 重新登录
version                            # 查看版本
```

### 4. 后续启动阶段

```
第二次启动 FutuOpenD
        ↓
读取 AppData.dat (上次登录保存的凭证)
        ↓
用凭证向富途服务器请求登录
        ↓
如果凭证有效 → 直接登录成功，无需验证码
如果凭证过期 → 重新需要验证码
```

**AppData.dat 的作用**:

- 保存登录后的 session/token
- 类似浏览器的"记住登录状态"
- 有时效性（取决于富途服务器策略）

## Telnet 端口设计原理

### 问题背景

**无头服务器（没有图形界面）无法输入验证码**

富途 OpenD 通常部署在 Linux 服务器上，但验证码登录需要人机交互。传统方案：
- 在本地打开富途牛牛 APP 输入验证码
- 服务器模式无法直接操作

### 富途的解决方案

```
┌─────────────────────────────────────────┐
│          Telnet 远程命令方案             │
└─────────────────────────────────────────┘

1. OpenD 启动一个 Telnet 服务器（22222 端口）
2. 用户可以通过 Telnet 远程发送命令
3. 这样即使在服务器上也能输入验证码
```

### 其他方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| Docker attach | 原生支持 | 仅限容器环境 |
| Telnet | 简单通用 | 明文传输 |
| HTTP API | 易集成 | 需要额外开发 |

**我们的实现**:

```bash
# 方案 1: expect 脚本自动化（适合 CI/CD）
./scripts/input-verify-code.exp 127.0.0.1 22222 358167

# 方案 2: bash 封装（简化使用）
./scripts/input-verify-code.sh 358167

# 方案 3: telnet 手动输入（适合调试）
telnet 127.0.0.1 22222
# 然后输入：input_phone_verify_code -code=358167
```

## 验证码 vs authcode

### 对比表

| 方式 | 原理 | 有效期 | 安全性 | 推荐场景 |
|------|------|--------|--------|----------|
| 验证码 | SMS 短信，每次登录可能需要 | 单次 | 中 | 首次配置、临时使用 |
| authcode | 富途生成的授权码，配置到 xml | 长期 | 高 | 生产环境、自动化部署 |

### authcode 原理

```
1. 在富途牛牛 APP 生成 authcode（绑定设备）
   路径：设置 → OpenD 设置 → 授权码

2. 配置到 FutuOpenD.xml:
   <authcode>你的 authcode</authcode>

3. OpenD 用 authcode 登录，无需验证码
   类似"API Key"的概念

4. 一次配置，长期有效
```

### 配置示例

```xml
<!-- 方式 1: 密码登录（需要验证码） -->
<login_account>15005162976</login_account>
<login_pwd>Brucelee0716</login_pwd>

<!-- 方式 2: authcode 免密登录（推荐） -->
<!-- <login_pwd>Brucelee0716</login_pwd> -->
<authcode>abcdef1234567890</authcode>
```

## 完整数据流

```
┌────────────────────────────────────────────────────────────────┐
│                        完整登录流程                              │
└────────────────────────────────────────────────────────────────┘

1. 配置文件          2. 启动进程           3. 服务器验证
   login_account       FutuOpenD             富途服务器
   login_pwd           读取配置              检查账号密码
       │                  │                     │
       │                  │── 登录请求 ───────→ │
       │                  │                     │
       │                  │← 需要验证码 ────────│── 发送 SMS
       │                  │                     │
       │   ┌──────────────┘
       │   │
       ↓   ↓
4. 用户输入验证码       5. 验证成功            6. 保存凭证
   手机收到 358167        FutuOpenD             AppData.dat
   ↓                     │── 提交验证码 ─────→ │ 保存 token
   输入命令               │                     │
   input_phone_          │← 验证成功 ──────────│
   verify_code           │
       │                 │── 开始服务 ─────→ 监听 11113
       ↓                 │
   Telnet 22222          │
                         ↓
                    7. 后续启动
                       读取 AppData.dat
                       用 token 登录
                       无需验证码（如果有效）
```

## 常见问题解答

### Q1: 为什么启动时不直接要验证码？

FutuOpenD 先尝试用密码登录，服务器判断需要验证码时才触发。如果 `AppData.dat` 有效，连密码都不需要。

```
启动 → 尝试密码登录 → 服务器返回需要验证码 → 等待用户输入
       ↓
   如果 AppData.dat 有效 → 直接登录成功
```

### Q2: 验证码会过期吗？

- **SMS 验证码本身**：通常 5-10 分钟
- **登录凭证（AppData.dat）**：几天到几周不等（由富途服务器决定）

### Q3: authcode 为什么更推荐？

- 一次配置，长期有效
- 不需要每次登录都等短信
- 适合自动化部署和 CI/CD
- 减少人为干预

### Q4: Telnet 安全吗？

- 只在本地 (`127.0.0.1`) 监听，外部网络访问不到
- 纯文本传输，但在内网环境没问题
- 如需更高安全性，可配合防火墙限制访问

### Q5: 如何判断是否需要验证码？

```bash
# 方法 1: 查看日志
tail -f Futu_OpenD_Centos7/*.log

# 方法 2: Telnet 连接测试
telnet 127.0.0.1 22222
# 如果返回 "等待验证码" 或类似提示，说明需要验证

# 方法 3: 直接尝试登录
# 如果需要验证码，富途服务器会自动发送 SMS
```

### Q6: 多个进程同时运行会怎样？

- 第一个进程会占用端口（11113、22222）
- 第二个进程会启动失败（端口冲突）
- 建议用 PM2 等工具管理，确保单实例运行
