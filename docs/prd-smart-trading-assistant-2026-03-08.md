# PRD：智能盯盘与行情助手

**文档类型**：产品需求说明（PRD）  
**状态**：Implemented  
**最后更新**：2026-03-10  

---

## 一、Context（背景与动机）

### 1.1 痛点

- 用户（jojo / slashhuang）持有或关注**美股、港股、比特币、黄金**等多类资产，希望**在交易日**能集中获取：
  - 持仓/自选标的的**当日行情**（涨跌、关键价位等）；
  - **条件满足时的提醒**（如涨跌超阈值、触及关键位、异常波动等），且希望**一次性汇总推送**，而不是零散多条。
- 除价格外，**国际新闻**（宏观、地缘、行业、重大政策等）也会影响决策，需要有人帮忙筛选并提示「需要警惕什么」「什么时候可能需要操作」。
- 现有能力：`skills/stock-monitor-1.3.0` 已支持基于 Yahoo Finance 的股价监控与 2%/1% 涨跌预警，但尚未覆盖：
  - 比特币、黄金等非股票资产；
  - 行情日报式汇总；
  - 可配置的「智能条件」与统一推送；
  - 新闻筛选与「操作/警惕」类建议。

### 1.2 机会

- 在 claw-family 内将「行情 + 条件盯盘 + 新闻 + 操作提示」整合为**智能投资助手**：在交易日主动或按节奏（如开盘前、盘中、收盘后）给用户一份**可操作的摘要**，并在一手通知中完成提醒与建议。

### 1.3 实现状态

- **Skill**: `skills/smart-trading-assistant/` 已创建（2026-03-10）
- **脚本**: `scripts/daily_brief.py` 支持行情获取、条件检查、新闻拉取、汇总输出
- **配置**: `config/assistant_config.json`（watchlist、conditions、schedule、news）、`config/trading_calendar.json`（美股/港股休市日）
- **状态**: `config/assistant_alert_state.json`（已 gitignore，记录条件触达冷却）

### 1.4 行业参考（智能投顾常见能力）

业内智能投顾/个人 AI 投研助手通常包含以下能力，供本 PRD 对齐与裁剪时参考：

| 能力 | 行业常见做法 | 本 PRD 的取舍 |
|------|----------------|----------------|
| **客户画像与风险评估** | 问卷评估风险偏好，动态画像 | **Out of Scope**：家庭场景暂不做正式 KYC/问卷；可选在配置中设「风险偏好/持仓偏好」用于建议语气 |
| **个性化资产配置** | 根据目标与风险给出配置方案 | **Out of Scope**：不自动生成配置方案，仅监控用户已有自选/持仓 |
| **交易执行与再平衡** | 阈值/定期再平衡，自动调仓 | **部分采纳**：仅做「组合偏离目标比例时的再平衡**提醒**」，不执行交易；可复用现有 `stocks_rebalance_monitor` |
| **实时监控与风险预警** | 7×24 追踪组合表现，市场异常与组合风险预警 | **采纳**：盯盘条件 + 新闻即「市场异常」；可补充「组合层面」预警（如单日组合回撤超 X%） |
| **绩效分析与报告** | 多维度业绩报告、与基准对比 | **后续迭代**：首版不做；可选后续增加周/月收益摘要 |
| **新闻与情绪** | 宏观/事件与行情关联、重大风险事件提示 | **采纳**：见 AC-9～AC-11 |
| **陪伴式/教练式服务** | 主动推送、操作建议、警惕非理性行为 | **采纳**：操作建议与警惕事项即轻量教练式提示；不做「客户行为监控」」 |

## 二、Goal（目标）

### 2.1 用户视角

- **交易日行情**：在交易日（按用户关注的交易日历：如美股、港股、部分节假日）能收到**基于自选/持仓的行情摘要**（涨跌、幅度、关键价位等），覆盖美股、港股、比特币、黄金等。
- **智能盯盘**：用户可配置多种**条件**（如涨跌超 X%、触及某价位、成交量异常、突破某均线等），当任一标的满足条件时，在**一次汇总**中收到提醒，避免刷屏。
- **新闻与风险提示**：能获取并筛选与持仓/自选相关的**国际新闻**（宏观、地缘、行业），并给出「需要警惕什么」「何时可能需要操作」的简要建议。
- **统一入口**：上述能力通过飞书与阿布人设统一呈现，形成「行情 + 盯盘 + 新闻 + 操作建议」的**智能助手**体验。

### 2.2 成功标准

- 用户在交易日能收到至少一次「行情摘要 + 条件触达提醒 + 相关新闻与操作/警惕建议」的汇总消息。
- 条件触达时，多条提醒合并为一次推送（一手通知）。
- 覆盖资产类型：美股、港股、比特币、黄金；可扩展至其它品种（如原油、外汇等）为后续迭代。

## 三、Acceptance Criteria（可验收条件）

### 3.1 行情摘要

| ID | 条件 | 说明 |
|----|------|------|
| AC-1 | 交易日触发 | 仅在用户配置的「交易日」发送行情摘要（如美股、港股交易日；可配置跳过 A 股/港股/美股节假日）。 |
| AC-2 | 标的范围 | 支持从配置中读取「自选/持仓」列表，包含：美股代码、港股代码、比特币（如 BTC-USD）、黄金（如 GC=F 或 GLD）等。 |
| AC-3 | 内容 | 摘要包含：标的名称、当前价、涨跌幅、相对基准/昨收、可选关键价位（如用户设置的观察价）。 |
| AC-4 | 频率 | 至少支持：开盘前汇总、收盘后汇总；可选盘中定时（如每 4 小时一次），由配置决定。 |
| **AC-4b** | **分类播报** | **行情摘要需按资产类别/市场分门别类展示**（如美股、港股、加密货币、期货/商品），便于用户快速浏览。 |

### 3.2 智能盯盘（条件提醒）

| ID | 条件 | 说明 |
|----|------|------|
| AC-5 | 条件类型 | 支持至少：涨跌超 X%、价格触及某价位（≥/≤）、可选：成交量或波动率异常（可后续迭代）。 |
| AC-6 | 配置方式 | 条件与标的在配置文件或 skill 配置中可编辑（如 `stocks_config.json` 或同级结构），无需改代码。 |
| AC-7 | 一手通知 | 同一周期内多个标的触发条件时，合并为**一条**提醒消息发送，而非每个标的一条。 |
| AC-8 | 去重与冷却 | 同一标的、同一条件在同一天内可配置「首次预警 + 续警」或冷却时间，避免重复刷屏。 |
| AC-8b | 组合再平衡提醒（可选） | 若用户配置了目标仓位/比例，当组合权重偏离目标超过阈值时，在汇总中给出「再平衡提醒」；可复用或对接现有再平衡监控脚本。 |
| AC-8c | 组合层面风险（可选） | 可选支持「组合单日回撤超 X%」或「组合波动率异常」时在汇总中给出风险提示，不要求首版必做。 |

### 3.3 新闻与操作建议

| ID | 条件 | 说明 |
|----|------|------|
| AC-9 | 新闻来源 | 支持从至少一种来源获取国际/财经新闻（如 RSS、第三方 API、爬虫等），需合规、可配置。 |
| AC-10 | 筛选 | 可根据用户关注的标的、行业或关键词做简单筛选，优先展示与持仓/自选相关的新闻。 |
| AC-11 | 建议 | 在推送中可包含 1～2 条「操作建议」或「警惕事项」（可由规则或简单模板生成，如「关注美联储表态」「某标的接近止损位」）。 |

### 3.4 集成与体验

| ID | 条件 | 说明 |
|----|------|------|
| AC-12 | 渠道 | 通过现有飞书通道推送，与阿布人设一致（中文、可带简要解读）。 |
| AC-13 | 可关闭/降频 | 用户可通过配置或指令临时关闭某类提醒或降低频率（如仅要盯盘不要新闻）。 |

## 四、Out of Scope（本次不包含）

- **实盘交易执行**：不下单、不接入券商交易接口，仅提醒与建议；不做自动再平衡调仓。
- **复杂量化策略**：不实现回测、多因子选股等，仅做「条件 + 行情 + 新闻」的展示与提醒。
- **客户画像与正式风险评估**：不做 KYC 问卷或动态风险画像；可选在配置中设简单偏好即可。
- **客户行为监控**：不监测用户「非理性交易行为」或做行为干预，仅做信息与建议推送。
- **税务优化（如 Tax-Loss Harvesting）**：不实现税务相关自动化建议。
- **A 股实时 Level-2 / 逐笔**：若后续支持 A 股，先以普通行情为主。
- **强制依赖某一家数据商**：设计上保留可替换数据源（如 Yahoo Finance、其它行情/新闻 API）的扩展能力。

## 七、可行性分析

### 7.1 技术可行性

| 维度 | 现状与结论 |
|------|------------|
| **行情与盯盘** | 现有 `stocks_monitor.py`、`stocks_rebalance_monitor.py` 已用 Python + Yahoo Finance Chart API（`query1.finance.yahoo.com/v8/finance/chart/{symbol}`）取价并做条件判断；同一接口支持美股、港股（如 `0700.HK`）、**BTC-USD**、**GC=F**（黄金期货）等，仅扩展配置与标的类型即可，**技术可行**。 |
| **一手汇总推送** | 当前监控脚本为「单次运行 → 打印/返回结果」；改为「单次运行收集行情 + 条件触达 + 可选新闻与再平衡结果，输出一份结构化摘要」即可。推送方式与现有一致：cron 触发脚本，**将输出通过 OpenClaw 的 cron 消息发到飞书**（见 stock-monitor SKILL 中的 `openclaw cron add ... --message "运行 ... 并把输出发给我"`），**可行**。 |
| **交易日历** | 需「仅交易日触发」：可用静态日历文件（如 JSON/CSV 维护美股、港股休市日）、或轻量 API（如 exchange-calendars、pandas_market_calendars 等开源库，或自建小表）。不依赖商业数据商，**可行**。 |
| **新闻与建议** | 新闻：RSS（如 Reuters、Bloomberg 部分频道）、或免费/低成本财经 API；筛选与「操作/警惕」首版用关键词 + 模板即可，无需 LLM。**可行**；若后续加 LLM 摘要为增强项。 |

**结论**：在现有技能栈（Python、Yahoo Finance、cron、飞书）上扩展即可实现 PRD 核心能力，无新语言或新基础设施依赖。

### 7.2 数据源可行性

| 数据需求 | 可选方案 | 可行性说明 |
|----------|----------|------------|
| **美股 / 港股 / 比特币 / 黄金** | Yahoo Finance Chart API（已用） | 同一 `chart?symbol=XXX` 支持 NVDA、0700.HK、BTC-USD、GC=F、GLD 等；非官方接口需注意**请求频率**（建议同一脚本内批量拉取、避免每分钟数百次），**可行**。 |
| **美股/港股交易日历** | 静态文件 + 年更 或 开源库 | 休市日可维护一份 JSON；或使用 `exchange_calendars`、`pandas_market_calendars` 等，**可行**。 |
| **国际/财经新闻** | RSS、News API、Alpha Vantage 新闻等 | 选 1～2 个稳定、合规的源；RSS 无 Key 限制但需解析与去重；部分 API 有免费额度。**可行**，需选定具体源并做稳定性与合规评估。 |

**结论**：行情与日历数据源明确、可落地；新闻源需在实现阶段选定并做一次合规与可用性确认。

### 7.3 资源与工期

| 项目 | 说明 |
|------|------|
| **人力** | 以技能形式在现有仓库内开发，可由 1 人（或 AI 辅助）增量完成：先行情 + 盯盘汇总，再新闻与建议，再可选再平衡/组合风险。 |
| **运行环境** | 与现有 OpenClaw 一致：需能跑 Python3、能访问 Yahoo 与新闻源、能通过 cron 触发并回传飞书；无额外服务器或付费服务硬依赖。 |
| **工期粗估** | 首版（行情摘要 + 盯盘条件 + 一手推送 + 交易日历）：约 1～2 周；加入新闻与建议：约 +0.5～1 周；再平衡/组合风险为可选增强。 |

**结论**：资源与工期在家庭/个人项目范围内可接受。

### 7.4 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **Yahoo Finance 限频或变更** | 取价失败或脚本被限流 | 控制调用频率（如按标的批量请求、cron 间隔不低于 5 分钟）；预留切换数据源设计（如 Alpha Vantage、yfinance 库等）。 |
| **交易日历不准** | 非交易日误发或交易日漏发 | 使用成熟开源日历库或人工维护的日历文件；上线前对当年主要节假日做一次核对。 |
| **新闻源不稳定或合规** | 推送失败或版权/合规问题 | 选用明确允许爬取/API 使用的源；首版可仅做标题+链接，避免全文爬取；失败时降级为「今日无新闻摘要」。 |
| **推送过于频繁** | 用户打扰 | 按 AC-13 支持按类型关闭/降频；汇总为一条消息，避免单条件一条。 |

**结论**：主要风险均可通过限频、可替换设计、降级策略与配置化缓解，无阻塞性风险。

### 7.5 综合结论

- **技术、数据源、资源与风险**均在可控范围内，**本 PRD 具备可行性**。
- 建议实现顺序：**行情 + 盯盘一手通知 + 交易日历** → **新闻 + 操作/警惕建议** → 可选 **再平衡提醒、组合风险**。

## 八、实现方向（建议）

### 8.1 Skill 结构

采用标准 OpenClaw skill 组织，位于 `skills/smart-trading-assistant/`：

```
skills/smart-trading-assistant/
├── SKILL.md                    # skill 说明文档（含 Heartbeat 任务章节）
├── _meta.json                  # skill 元数据
├── config/
│   ├── .gitignore              # 忽略运行时状态
│   ├── assistant_config.json   # 主配置（watchlist、conditions、schedule、news）
│   ├── trading_calendar.json   # 交易日历（美股/港股休市日）
│   └── assistant_alert_state.json  # 运行时状态（已 gitignore）
└── scripts/
    └── daily_brief.py          # 日报脚本（行情 + 条件 + 新闻汇总）
```

### 8.2 配置说明

#### 8.2.1 watchlist 配置

标的列表，支持以下格式：

| 市场 | 代码格式 | 示例 |
|------|----------|------|
| 美股 | 直接写代码 | `NVDA`, `HIMS`, `CLSK`, `FUTU`, `SMCI`, `COIN` |
| 港股 | 代码 + `.HK` | `0700.HK`, `9868.HK` (小鹏), `2015.HK` (理想) |
| A 股 | 代码 + `.SS` / `.SZ` | `600519.SS` (茅台) |
| 加密货币 | `XXX-USD` | `BTC-USD`, `ETH-USD` |
| 期货/商品 | Yahoo 代码 | `GC=F` (黄金), `CL=F` (原油) |

#### 8.2.2 其他配置

- **conditions**：盯盘条件（涨跌超 X%、触及价位），支持 cooldown_hours 冷却
- **schedule**：报告时段、交易日历区域、禁用区块
- **news**：RSS 源、关键词筛选、最大条数
- **rebalance**：可选，指向再平衡配置路径

### 8.3 执行节奏

由 cron / 定时任务在交易日按配置调用脚本：

```bash
python3 skills/smart-trading-assistant/scripts/daily_brief.py \
  skills/smart-trading-assistant/config/assistant_config.json
```

输出为 Markdown，通过 OpenClaw cron 的 `--message` 参数发到飞书。

### 8.4 新闻与建议

首版采用规则/关键词 + 模板生成「警惕/操作」建议；后续若有需要再考虑轻量 LLM 摘要。

### 8.5 再平衡与组合风险

若有「组合目标比例」配置，可复用或对接 `skills/stock-monitor-1.3.0/scripts/stocks_rebalance_monitor.py` 的输出，并入同一汇总推送。
